thoroughly; the github doesn't reflect the update or [is not] at a state

(315 tests, exit 0 at the time). The rot was entirely in the **documentation

`VersionConsistency` suite only asserted the README hero `Download vX.Y.Z`
CTA, the DMG filename, the CHANGELOG head, and the SECURITY major series, so

## Reusable learnings (extracted to memory this session)

- **A guard only protects what it asserts.** When a "ship" leaves docs stale,
  the permanent fix is (1) reconcile the docs AND (2) widen the guard so the
  unasserted-drift class cannot recur, anchoring human-written numbers to
  machine ground truth (the live test registry). Saved as trap #5 in
  `reference_testing_gotchas` and in `reference_multipaste_app`.
- **The DocConsistency pattern** (reusable for any repo): assert stated
  counts == a live registry count; assert table rows sum to the stated
  total; assert every CLI flag / relative link is documented / resolves.
- **Scanner blind spot banked:** a link/flag scanner sees literal `](x)` and
  `--x` inside inline-code backticks. `readmeRelativeLinksResolve` skips a
  `](...)` match fully wrapped in backticks (before/after char is a
  backtick), so prose ABOUT link syntax does not false-positive.

## Hardening II (follow-up, commit `5d5f8d5`, 323 -> 326)

Round I fixed the docs and guarded the surfaces it asserted. Applying the
same lesson (a guard only protects what it asserts) to the surfaces still
open surfaced three more, all closed with new `DocConsistency` guards
(all mutation-proven to redden EXACTLY their own guard):

- `coverageRowCountsMatchPerSuiteRegistry`: each coverage row's count must
  equal the LIVE per-suite registered count. The registry names every case
  `Suite/testName`, so per-suite live counts are tallied from the prefix.
  This is **strictly stronger** than `coverageTableRowsSumToTotal`: a
  compensating miscount (one suite +1, another -1) keeps the global total
  right and slips past the sum guard, but the per-suite guard catches it.
  Proof: mutating ClipboardItem 11->10 AND SnippetMatcher 11->12 (total
  stays 326) reddened ONLY this guard (1 failed, 325 passed), with the two
  global guards staying green. Also fails loudly on a missing suite row or
  a stale/renamed row (the row-prefix set must equal the live suite set).
- `everyCanonicalDocRelativeLinkResolves`: the round-I dead-link guard only
  scanned README; this scans the OTHER canonical navigational docs
  (SECURITY, CONTRIBUTING, the PR template, RELEASING, the roadmap). It uses
  a robust odd-backtick-count inline-code skip, not the flanking-char check,
  so a `](x)` wrapped in a larger inline-code span (e.g. CHANGELOG's
  `` `[link](url)` ``) never false-fails. CHANGELOG is excluded on purpose:
  append-only record, link-like fragments are illustrative prose. Proof:
  injecting `[ghost](ghost-does-not-exist.md)` into CONTRIBUTING reddened
  ONLY this guard.
- `filesTreeTestAndSuiteCountsMatchRegistry`: pins the Files-tree
  "N tests across M suites" line (the only place the suite count is written,
  and it uses the bare word "tests" so the "N unit tests" guard skipped it)
  to the live case count and the live distinct-suite count. Proof: mutating
  it to "999 tests across 27 suites" reddened ONLY this guard.

New reusable learnings banked to memory this round:
- **A global sum guard is weaker than a per-key guard**; the compensating
  mutation that survives the sum but dies under the per-key check is the
  proof the per-key guard adds real coverage (see [[reference_test_mutation_teeth]]).
- **The odd-backtick-count rule** is the robust way to skip inline-code
  matches; the flanking-char shortcut misses links wrapped in a larger span.
- Extracted a portable `doc-linkcheck` CLI + skill from the link guard (the
  universal, repo-agnostic half of this pattern).

Deliberately NOT built: a reverse-flag guard ("every documented `--flag` is
a real app flag"). README legitimately documents three flag namespaces (the
app, the `clip-pin` helper, and brew/xcode packaging), indistinguishable by
syntax, so such a guard would false-fail on a legitimate edit. A fragile
guard is worse than none (guards are assistive, not restrictive).

No `Sources/` change: docs+tests only, no version bump, shipped v2.4.2
artifacts unaffected. `make build` + `make verify-app`: exit 0, app verified
universal/signed/version-consistent.
