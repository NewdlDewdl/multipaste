## Reusable learnings (extracted to memory this session)

- **A guard only protects what it asserts.** When a "ship" leaves docs stale,
  the permanent fix is (1) reconcile the docs AND (2) widen the guard so the
  unasserted-drift class cannot recur, anchoring human-written numbers to
  machine ground truth (the live test registry). Saved as trap #5 in
  `reference_testing_gotchas` and in `reference_multipaste_app`.
- **The DocConsistency pattern** (reusable for any repo): assert stated
  counts == a live registry count; assert table rows sum to the stated
  total; assert every CLI flag / relative link is documented / resolves.
- **Scanner blind spot banked:** a link/flag scanner sees literal `](x)` and
  `--x` inside inline-code backticks. `readmeRelativeLinksResolve` skips a
  `](...)` match fully wrapped in backticks (before/after char is a
  backtick), so prose ABOUT link syntax does not false-positive.
