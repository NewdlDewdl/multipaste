## Ship sequence (exact; details in docs/RELEASING.md)

1. Gates on the ship branch: `make test` (expect 302), `make
   plaintext-smoke-test`, `make smoke-test`, `make verify-app`; read exit
   codes.
2. RE-CHECK the DMG hash matches the table above.
3. `git checkout main && git merge --no-ff review/v2.4.0-fable-audit -m
   "v2.4.0: paste as plain text (⇧↩) + snippet-trigger fix, adversarially
   reviewed"`; re-run `make test && make verify-app` ON MAIN.
4. `git push origin main` and push both feature/review branches for
   history.
5. Extract CHANGELOG 2.4.0 body to a notes file; `gh release create
   v2.4.0 dist/Multipaste-2.4.0.dmg --title "v2.4.0: paste as plain
   text" --notes-file <file>`; verify with `gh release view v2.4.0
   --json tagName,assets` and confirm the uploaded asset's sha256 by
   downloading it back (CoVe: verify the thing itself).
6. Cask: version `2.4.0`, sha256 `59522ee5...73f1d`; commit `multipaste
   2.4.0`; push.
7. `brew update && brew upgrade --cask multipaste`; verify installed
   Info.plist says 2.4.0, app relaunched, log fresh, paste works with NO
   Accessibility re-grant (stable codesign DR).
8. Live manual checklist (5 steps) from the end of
   `docs/reviews/v2.4.0-FINDINGS.md`; synthetic ⌘⇧V works on macOS 26
   for headless picker opening.
9. Confirm the update pipeline: `curl -s
   https://api.github.com/repos/NewdlDewdl/multipaste/releases/latest`
   tag == v2.4.0 (this is what the in-app UpdateChecker reads).
