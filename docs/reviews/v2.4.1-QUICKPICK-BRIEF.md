# Fix brief: ⌘1-9 quick-pick should target the RECENT rail (v2.4.1)

Self-contained anchor for the session that builds and ships this fix.
Every claim below was CoVe-verified live on 2026-07-04 against the
installed 2.4.0 build and the source at `main` = `5a6b2a9` (tree clean,
302 tests green). The generic ship procedure lives in `docs/RELEASING.md`.

## Rohin's report (verbatim intent)

"fix the current behavior where command+1 pastes the 1st recent item
besides the pinned items which doesnt align with the labels for example
it puts command + 1 at the first pinned item"

## Verified ground truth (IMPORTANT: the premise needed correction)

The report reads like a label/action desync. Live verification shows the
current build is SELF-CONSISTENT; the real problem is the design:

- Picker: rows are `filtered = store.search(query)` which is pinned-first
  (`HistoryStore.search` -> `sortedForDisplay()`, HistoryStore.swift:171).
  Badge = `"⌘\(index + 1)"` where index = row (PickerWindow.swift:640,
  wired at :337 `ItemCellView(item: filtered[row], index: row, ...)`).
  Handler = `filtered[digit - 1]` (PickerWindow.swift:395-403). Live test
  2026-07-04: with 4 pinned items, ⌘1 pasted the FIRST PINNED row, which
  is exactly what its badge said (log `pickAndPaste[single]` + pasteboard
  string matched row 1).
- Menu: `sortedForDisplay().prefix(9)` gets keyEquivalents ⌘1-9 in the
  same pinned-first order (MenuBarController.swift:166-180); handler uses
  `representedObject` id, so it is also internally aligned. Live test:
  menu ⌘1 pasted the same first pinned item.
- No global digit hotkeys exist (HotKeyManager registers only the picker
  hotkey).

So: labels and actions AGREE today, but both spend ⌘1-⌘4 on Rohin's 4
pinned items. Pinned items are the STABLE rail (they never move; no
speed win from digits), while the recents rail is where quick-pick
matters. Both readings of the report converge on the same end state.

## Required end state

**⌘1-9 (picker AND menu) target the first 9 UNPINNED items in display
order. Pinned rows carry no digit badge / no keyEquivalent.** Labels
follow the action everywhere:

1. Picker: the Nth unpinned row in the current `filtered` list shows
   `⌘N` (N <= 9); pinned rows show no ⌘ badge. ⌘digit pastes the Nth
   unpinned FILTERED row (filter active = digits target visible unpinned
   matches). Flavor behavior unchanged (`effectiveFlavor(shiftPressed:
   false)` as today, PickerWindow.swift:399-401).
2. Menu: keyEquivalent "1"-"9" go on the unpinned recent items only, in
   the same order the menu displays them; pinned menu rows get "".
3. Hint bar text `⌘1–9 quick` stays as is.
4. The multi-paste MARK badge (filled accent capsule, `markIndex`,
   PickerWindow.swift:337-338 and :681) is a SEPARATE mechanism; do not
   touch its numbering.

## Architecture requirement (repo convention, see PasteFlavor for the model)

Put the DECISION in pure Core so it is unit-testable, keep AppKit thin:

- Add a Core policy (suggested: extend HistoryStore or a small
  `QuickPick` type): given a display-ordered item list, return the
  digit->item mapping (first 9 unpinned) and, per row, its digit label
  (nil for pinned / beyond 9). One source of truth consumed by BOTH the
  picker badge, the picker handler, and the menu keyEquivalent loop, so
  the three surfaces cannot drift again.
- Unit tests for the policy: all-pinned (no digits), no-pinned (digits =
  rows 1-9), mixed (digits skip pinned), >9 unpinned (10th gets none),
  filter subsets, empty list. Mutation-teeth per repo rule
  (reference_test_mutation_teeth): e.g. mutate the policy to number
  pinned-first, confirm exactly the new tests redden, revert.
- The AppKit wiring gets the live gate: after install, use the
  headless recipe below.

## Live verification recipe (proven this session; docs/RELEASING.md has the full version)

- `scripts/pbtool.swift` (in-repo): `swiftc -O scripts/pbtool.swift -o
  /tmp/pbtool`; modes `rich <text>` / `file <path>` / `info`. The
  general pasteboard is the oracle; pair with the log
  (`pickAndPaste[single|combined]`).
- Synthetic ⌘⇧V opens the picker (System Events key code 9 cmd+shift);
  key code 18 + command = ⌘1. PITFALLS (all hit this session):
  screencapture makes the panel resign key and hide (screenshot in a
  SEPARATE picker session); pinned items sort first so know your rows;
  type a filter to make row targets deterministic; synthetic ⌥↩ never
  registers (use ⌥⌘A for marks).
- Post-fix acceptance: with >=1 pinned + >=1 unpinned item, ⌘1 must
  paste the FIRST UNPINNED item (pasteboard string proves it); a
  badge screenshot must show ⌘1 on that row and NO ⌘ badge on pinned
  rows; menu ⌘1 (menu open) must paste the same item.
